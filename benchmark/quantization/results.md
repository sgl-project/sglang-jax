# MiMo-V2-Flash FP8 Quantization Benchmark Results

- Model: MiMo-V2-Flash (`hidden_size=4096, num_attention_heads=64, num_key_value_heads=4/8, head_dim=192, v_head_dim=128`)
- Quantization: FP8 (float8_e4m3fn), block-size=128, weight-only (no activation quantization)
- Kernel: BF16 使用 `lax.dot_general`（XLA 原生），FP8 block-128 使用 Pallas `blockwise_kernel`
- Platform: TPU v6e (2x2, 4 chips, single-host)
- 计时方式: 从 JAX profiler trace 中提取 kernel-level `device_duration_ps`，匹配 `tf_op` 中的 scope name
- Tokens: 1, 64, 128, 256, 512, 1024, 2048, 4096

---

## HLO 观察到的实际 Quantized Matmul Cases

以下 3 个 case 来自模型实际编译产生的 HLO，均为 TP=1 场景：

| HLO Kernel | 计算 (X x W^T -> Out) | 对应 Projection | BF16 (ms) | FP8 (ms) |
|---|---|---|---|---|
| `quantized_matmul_kernel_128_256_128` | `bf16[2048, 4096] x f8e4m3fn[768, 4096]^T -> bf16[2048, 768]` | k_proj (full attn, tokens=2048) | 0.020 | 0.706 |
| `quantized_matmul_kernel_128_256_128` | `bf16[2048, 4096] x f8e4m3fn[1024, 4096]^T -> bf16[2048, 1024]` | v_proj (SWA, tokens=2048) | 0.025 | 0.941 |
| `quantized_matmul_kernel_64_256_128` | `bf16[64, 4096] x f8e4m3fn[768, 4096]^T -> bf16[64, 768]` | k_proj (full attn, tokens=64) | 0.002 | 0.004 |

> kernel 名中的数字为 Pallas tiling 参数 (block_m, block_n, block_k)。Weight 以转置形式 `[out, in]` 存储。

---

## 结论

**FP8 block-128 在 TPU v6e 上全面落后于 BF16**。

根因：`tuned_block_sizes.py` 中 TPU v6 **仅有 INT8 调优条目，没有 FP8 (float8_e4m3fn) 条目**。所有 FP8 配置均触发 `Couldn't find tuned sizes for the quantized matmul kernel` 警告，Pallas blockwise kernel 使用默认 block sizes，性能极差。

| 观察 | 说明 |
|------|------|
| BF16 TFLOPS 可达 500-3200 | `lax.dot_general` 充分利用 MXU，接近 TPU v6e 峰值 |
| FP8 TFLOPS 封顶 ~18 (TP=1) | 未调优 Pallas kernel 吞吐极低 |
| FP8 TP=4 可达 ~146 TFLOPS | TP 越高每设备 weight 更小，Pallas 开销占比下降 |
| tokens 越大差距越大 | BF16 线性 scale，FP8 Pallas 未调优无法 scale |

## 测试的计算规模

benchmark 计算: `X @ W`，其中 X = `[tokens, in_features]`，W = `[in_features, out_features]`。

Column parallel (q/k/v_proj): 沿 `out_features` 维度切分 weight，每设备 W = `[in, out/tp]`。
Row parallel (o_proj): 沿 `in_features` 维度切分 weight，每设备 W = `[in/tp, out]`。

### 本次测试的 Projection

| Projection | 类型 | 逻辑 W | TP=1 每设备 | TP=2 每设备 | TP=4 每设备 |
|---|---|---|---|---|---|
| q_proj | column | `[4096, 12288]` | `[4096, 12288]` | `[4096, 6144]` | `[4096, 3072]` |
| k_proj (full, kv=4) | column | `[4096, 768]` | `[4096, 768]` | `[4096, 384]` | TP=4 跳过 (768/4=192 不整除 128) |
| v_proj (SWA, kv=8) | column | `[4096, 1024]` | `[4096, 1024]` | `[4096, 512]` | `[4096, 256]` |
